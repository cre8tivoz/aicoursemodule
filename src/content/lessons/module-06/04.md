# Capstone alignment: what you will submit

Module 06 turns **intent** (brief), **execution** (prompts and loops), and **judgement** (review) into something **assessable**. This lesson is a **pre-flight** — not the build itself.

| Document | Role |
| -------- | ---- |
| [`assessments/capstone.md`](../../assessments/capstone.md) | **Authority** — requirements, checklist, marking table |
| [Module 07, Lesson 01](/lessons/module-07/01) | **Build walkthrough** — folders, structure, deploy, Lighthouse |
| This lesson | **Alignment** — trace your brief to what assessors will check |

Complete this lesson **before** the Module 06 check-in. After the check-in, open Module 07 and start building.

## Map brief → marking rows

Your **brief** (Lesson 01) should already name pages and purpose. Trace each section to a capstone row:

| Brief element | Capstone row it feeds |
| ------------- | --------------------- |
| **Goal** | Structure / UX — visitor understands “what this is” quickly |
| **Audience** | Structure / UX + Content — tone, reading level, mobile layout |
| **Pages/sections** | Structure / UX — 1–3 HTML docs, working nav |
| **Must-haves** | Each becomes one line in your self-audit below |
| **Accessibility intent** | Accessibility — semantics, keyboard, contrast, Lighthouse ≥ 90% |
| **Non-goals** | Scope discipline — extra pages risk broken layout or missed checks |

If your brief promised five pages, **narrow it now** — the capstone allows **1–3** pages only.

## Linking your Module 06 work to the rows

- **Structure / UX** — Brief + Lesson 02 prompts (nav, hierarchy, mobile); Lesson 03 loop catches regressions.
- **Accessibility** — Module 03 checklist + Module 05 literacy + Lighthouse on the **deployed** URL.
- **Visual design** — Palette from your brief; avoid random colours that fail contrast.
- **Deployment** — Live URL you can justify in one sentence.
- **Content** — Your words; respect copyright on images and text.

If any row is **NS** today, write **one concrete action** (not “try harder”) to move it to **S**.

## Do this — submission dry run

List exactly what you will paste into your submission form:

- **URL** line
- **Lighthouse** line (score, date, Chrome version)
- Optional **repo** line
- **One sentence** on hosting choice (cost, privacy, IT policy)
