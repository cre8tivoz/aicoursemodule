# Capstone alignment: what you will submit

The capstone is **work-integrated**: you ship a **live site** and fill an **evidence pack** — not an essay.

| Document | Role |
| -------- | ---- |
| [`assessments/capstone.md`](../../assessments/capstone.md) | **Submission template** — Parts A–D |

| This lesson | **Alignment** — brief → marking rows before you code |

## Map brief → marking rows

| Brief element | Capstone row it feeds |
| ------------- | --------------------- |
| **Goal** | Structure / UX |
| **Audience** | Structure / UX + Content |
| **Pages/sections** | Structure / UX (1–3 HTML docs) |
| **Must-haves** | One proof line each in Part B |
| **Accessibility intent** | Accessibility row + Lighthouse |
| **Non-goals** | Scope — cut before build |

If your brief promised five pages, **narrow it now**.
