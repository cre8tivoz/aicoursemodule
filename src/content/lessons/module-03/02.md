# Connecting LLMs — IDE extensions, Copilot, Claude, Gemini, and Gemini CLI

Lesson 1 said the **IDE + folder** is your home base. This lesson is **practical**: how to **connect** large language models (LLMs) from major families—**OpenAI-backed** tooling, **Anthropic Claude**, **Google Gemini**—so you are not stuck copy-pasting from a browser for every line.

## A. IDE extensions (VS Code, Cursor, and similar)

These steps are **generic**. Your editor may label menus slightly differently.

### 1. Open the marketplace

- **VS Code / VSCodium:** **Extensions** view (`Ctrl+Shift+X` / `Cmd+Shift+X`), or the square-blocks icon in the sidebar.  
- **Cursor:** **Extensions** work similarly (Cursor is VS Code–compatible for many extensions).

### 2. What to search for (official publishers)

Install only **extensions from publishers you recognise**. Examples people often use (verify names in the marketplace today):

| Family | What to look for | Notes |
| ------ | ---------------- | ----- |
| **OpenAI / Microsoft path** | **GitHub Copilot** (publisher: **GitHub** / Microsoft ecosystem) | Uses model-backed assistance in the editor; **subscription or trial** rules apply—read the product page. Often the main “OpenAI-class” experience **inside** VS Code. |
| **Anthropic Claude** | Search **“Claude”** or **“Anthropic”** | Anthropic publishes (or partners on) extensions that vary by region and product name. Read install counts and **publisher** carefully. |
| **Google Gemini** | **Gemini Code Assist** or other **Google**-published Gemini extensions | May require **Google account** sign-in; enterprise vs personal differs. |

**Install flow (typical):**

1. Click **Install** on the extension.  
2. Reload the editor if asked.  
3. Open the command palette (`Ctrl+Shift+P` / `Cmd+Shift+P`) and run the extension’s **sign-in** command if prompted.  
4. Complete **OAuth** in the browser window (Google, GitHub, Anthropic, etc.).  
5. Open your **`my-static-site`** folder and try a **small** request (“explain this file”) to confirm the tool sees your workspace.

### 3. Cursor-specific note

**Cursor** ships with **integrated** AI features; you may rely less on separate “Claude” or “OpenAI” extensions. Still follow **sign-in**, **model choice**, and **privacy** settings inside Cursor’s own UI. The **point** is the same: **work against real files**.

## B. Gemini CLI (terminal) — free tier with a Google account

**Gemini CLI** is Google’s **terminal** agent for Gemini. It is **open source** (Apache 2.0) and documented on GitHub: [`google-gemini/gemini-cli`](https://github.com/google-gemini/gemini-cli). The npm package is **`@google/gemini-cli`**.

### Prerequisites

- **Node.js 18+** and npm (or use `npx` / Homebrew per docs).  
- A **Google account** for **Sign in with Google** (OAuth).

### Install (common paths)

**Option 1 — run without global install**

```bash
npx @google/gemini-cli
```

**Option 2 — install globally**

```bash
npm install -g @google/gemini-cli
```

**Option 3 — macOS/Linux (Homebrew)**

```bash
brew install gemini-cli
```

Full detail: [installation docs](https://github.com/google-gemini/gemini-cli/blob/main/docs/get-started/installation.md) on the repo.

### First run and auth

Start the CLI:

```bash
gemini
```

Choose **Sign in with Google** when prompted and finish the browser flow. For many personal accounts, Google documents a **free tier** (rate limits apply—see current terms on Google’s site). No API key is required for that OAuth path if you use **Sign in with Google**.

> **Not the IDE, but paired with it**  
> Use the terminal **next to** VS Code/Cursor: CLI for questions and tasks; **IDE** for clicking files and diffs. Same project folder.

## C. Browser chat — use it, but know the role

You can always open **chat.openai.com**, **claude.ai**, or **Gemini** in the browser. Good for **Explain** and rough **Generate** when you are away from the project.

**Iteration in the browser** feels fast: you type, you get an answer, you type again. That habit can hide three problems for static sites:

1. **No file truth** — Which paragraph is “current” if you have ten chat turns?  
2. **No project context** — The model may forget your folder structure unless you re-paste it.  
3. **No clean diff** — You cannot `git diff` a chat.

**This course’s point:** shipping a static site means **files on disk** and a **repeatable loop**. Extensions + CLI bring models **closer** to that loop. Browser chat is **supplemental**, not the definition of “done.”

**Pedagogy:** we still want you comfortable in a browser—questions, tutorials, quick drafts—so you are not afraid of the web. The **graded professional habit** for this subject is: **open the IDE**, open the **workspace folder**, then use AI **against those files**.

## Troubleshooting (common snags)

- **“Extension installed but nothing happens”** — You usually still need **Sign in**. Open the command palette and search the extension name + **Sign in** or **Activate**.  
- **`npm` / `npx` not found** — Install **Node.js LTS** from nodejs.org (or your package manager), restart the terminal, try again.  
- **Gemini CLI opens browser then fails** — Check clock sync, VPN rules, and whether a **corporate proxy** blocks Google OAuth. Ask IT with a screenshot **without** sensitive data.  
- **Two extensions fight** — Disable one assistant at a time; keep the workflow simple until you are confident.

## Strengths and boundaries (short)

- **Hallucination:** models invent plausible markup—**verify** against your brief (Lesson 4 checklist).  
- **Policy:** employer may block certain extensions or cloud sign-ins—**ask IT**.  
- **Cost:** Copilot and others may need **paid** plans—read before you rely on them.  
- **Region:** not every extension is available in every country.  
- **Model updates:** behaviour changes month to month—**re-test** old prompts occasionally.

## Key takeaways

- **Extensions:** install from **trusted publishers**; **sign in**; confirm the tool sees your **workspace**.  
- **Gemini CLI:** `npx @google/gemini-cli` or `npm install -g @google/gemini-cli`; **`gemini`** then **Sign in with Google** for the common free-tier path.  
- **Browser chat** is fine for learning; **professional practice** in this course centres on **IDE + files**.
