# Hand-coding, AI-first, and hybrid paths

Online debates love camps. Professional work is **trade-offs**: goal, time, risk, and what you must maintain next month.

| Path | Strength | Cost |
| ---- | -------- | ---- |
| **Hand-coding first** | Deep literacy; works offline | Slower first page |
| **AI-first** | Fast blank-page relief | Hidden mistakes without review |
| **Hybrid** (most adults) | Pragmatic mix | Needs honesty about what you skip |

**Hybrid examples:** hand-write structure, AI drafts copy · AI drafts, you fix headings · [freeCodeCamp](https://www.freecodecamp.org/) in parallel for reading skills.

Higher stakes (brand, legal, accessibility promises) → **more review**, any path.

## Do this — pick and justify (15 minutes)

**Scenario:** Personal portfolio live by **Monday morning**. One weekend. You are not a web developer by title.

Write **6–8 sentences**:

1. Path: **hand-first**, **AI-first**, or **hybrid** (define in one line)  
2. **Two reasons** tied to the scenario (time, credibility, learning)  
3. **One risk** of your path + **one mitigation** you will actually do  

Avoid thin answers like “AI is faster” with no verification plan.

## Example — weekend portfolio (sketch)

**Hybrid:** brief Saturday morning → AI skeleton Saturday afternoon → **manual** heading and link fixes → mobile test Sunday → friend read for clarity. Risk: vague copy. Mitigation: read contact block aloud before deploy.

## Check

- [ ] Your justification names **stakes**, not slogans  
- [ ] You included a **concrete** review step
