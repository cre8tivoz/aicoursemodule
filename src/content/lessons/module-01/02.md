# AI as accelerator, not a substitute for judgement

AI can draft HTML and copy **fast**. It can also sound **confident when wrong**. Your job is to **compress draft time**, not **compress responsibility**.

**Drafting** = getting words and structure on the page.  
**Deciding** = what is accurate, on-brand, safe, and usable for real visitors.

## Risk map (read once)

| Area | What goes wrong | Quick check |
| ---- | --------------- | ----------- |
| **Truth** | Wrong dates, prices, contact details | Read aloud what matters |
| **Structure** | Skipped headings, missing landmarks | Tab + glance at `h1`–`h3` |
| **Privacy** | Pasting secrets into cloud tools | Policy + common sense |
| **Tone** | Generic or excluding language | Would your audience recognise this voice? |

## Example — hero heading (1 minute)

AI returns a cheerful hero with the page topic as `<h2>` because it guessed the logo was “top.” If “Volunteer clean-up day” is the real topic, promote it to `<h1>`. Fix button text: **Register**, not **Click here**. Speed from AI; **judgement** from you.

## Do this — three pre-publish checks (10 minutes)

List **three things** you would **always** verify before publishing AI-written HTML for a **public** work page. For each, add **one sentence** why it matters.

**Format:**

1. **___** — Because …  
2. **___** — Because …  
3. **___** — Because …  

**Ideas:** heading structure · contact details · link text · contrast at a glance · third-party embeds.

**Stretch:** Add a fourth about **tone** and when you would ask a human reviewer.

## Say this in a meeting

> “AI helps me draft faster; I verify structure and facts before publish.”

That sets realistic expectations about review time.

## Check

- [ ] Your three checks are **specific** enough to do next week  
- [ ] You can separate **draft speed** from **accountability**
