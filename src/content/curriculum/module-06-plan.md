# Module 06 — Prompting as design: briefs, constraints, review loops

**Course outcomes:** CO-04, CO-05

## Module objectives (AQF 5)

1. Write a project **brief** (audience, pages, tone, must-haves) usable by a human or AI collaborator.
2. Convert the brief into **prompts** with constraints and examples of “done.”
3. Run an **iteration loop**: generate → review against CO-03 criteria → refine.

## Prerequisites

- Modules 01–05 (especially M01 creative brief, M03 checklist, M05 semantics).

## Lessons (4)

### Lesson 01 — From vague idea to a one-page brief

| Phase | Content |
| ----- | ------- |
| **Concept** | Sections: **goal**, **audience**, **pages/sections**, **tone**, **must-haves**, **non-goals**, **accessibility intent**. |
| **Example** | Fill-in template for “hobby micro-site” vs “dream business landing.” |
| **Practice** | Complete the template for your own capstone topic (draft). |
| **Summary** | The brief is the contract with yourself and the AI. |
