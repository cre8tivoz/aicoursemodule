# Module 02 — IDEs and a sane project workflow

**Course outcomes:** CO-02

## Module objectives (AQF 5)

1. Set up a minimal project folder and open it in an IDE with confidence.
2. Use core IDE affordances that speed static site work (navigation, multi-file edit, formatting, optional Git).
3. Describe how IDE integration lowers the cost of trying ideas compared to notepad-style editing.

## Prerequisites

- Module 01 complete conceptually (why static + creative brief mindset). **No formal hurdle.**

## Lessons (4)

### Lesson 01 — Your project folder is the product

| Phase | Content |
| ----- | ------- |
| **Concept** | One folder = one site: `index.html`, `css/`, `images/`. Paths matter (`href`, `src`). |
| **Example** | Diagram: `site/index.html` linking to `css/style.css` and `images/logo.png`. |
| **Practice** | Sketch a folder tree for a 2-page hobby site (names only). |
| **Summary** | Structure first; reduces confusion when AI generates multiple files. |

**File:** [Module 02 Lesson 01](/lessons/module-02/01) (Phase 3 — hands-on, ≤800 words)

---

### Lesson 02 — Opening the IDE and the “workspace”

| Phase | Content |
| ----- | ------- |
| **Concept** | **Workspace** = root folder. Modern IDEs (VS Code, Cursor, OpenCode, etc.) index files for search and AI context. |
| **Example** | Open folder → verify file tree → open `index.html`. |
| **Practice** | Checklist: install editor, open folder, create empty `index.html`, save. |
| **Summary** | Ease of use starts with treating the folder as the single source of truth. |

**File:** [Module 02 Lesson 02](/lessons/module-02/02)

### Lesson 03 — Edit, preview, search, format

| Phase | Content |
| ----- | ------- |
| **Concept** | **Live preview** (built-in or extension), multi-cursor, find/replace across files, formatter for HTML/CSS. |
| **Example** | Change a title in HTML and see preview update; find all `class="btn"` in project. |
| **Practice** | List three IDE actions that save time vs a basic text editor. |
| **Summary** | Friction down → more iterations → better learning and output. |

### Lesson 04 — Optional Git snapshot (concept only)

| Phase | Content |
| ----- | ------- |
| **Concept** | Version control = undo history for adults; not required for the capstone but useful. |
