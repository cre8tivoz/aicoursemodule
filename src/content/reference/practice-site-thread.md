# Practice site thread

One folder, many modules. You are not building six separate toy sites — you are growing **`my-static-site`** from Module 02 into something you can **read**, **fix**, and eventually **replace or upgrade** for the capstone.

## The thread

| Module | What you do with `my-static-site` |
| ------ | ----------------------------------- |
| **M02** | Create the folder: `index.html`, `about.html`, `css/style.css`, `images/` |
| **M03** | Connect AI to the **workspace**; save generated edits back into these files |
| **M05** | Open `index.html` in the editor and **fix structure** (landmarks, headings) |
| **M06** | Write a **brief** — can be for this practice site **or** a new capstone topic |
| **M07** | Ship the **capstone** (1–3 pages). You may rebuild from scratch on a new topic — the habits stay the same |

## Why bother?

- You stop re-explaining your folder layout in every new chat.
- You see how **small fixes** (one `h1`, one `alt` text) change Lighthouse and readability.
- When something breaks after deploy, you know **which file** to open.
